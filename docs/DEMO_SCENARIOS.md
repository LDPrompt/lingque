# 灵雀演示场景

以下是 2 个可现场演示的典型场景，展示灵雀的核心能力。

---

## 场景一：电商数据采集

**展示能力**：浏览器自动化 + 数据抓取 + 结构化输出

### 演示步骤

1. 在飞书中对灵雀说：

```
帮我打开京东，搜索"蓝牙耳机"，采集前 10 个商品的名称和价格
```

2. 灵雀自动执行：
   - `browser_open("https://www.jd.com")` 打开京东
   - `browser_type` 在搜索框输入关键词
   - `browser_click` 点击搜索按钮
   - `browser_scroll_collect` 滚动采集商品列表
   - 输出结构化表格

3. 演示要点：
   - RPA 级别的鼠标移动和人类化输入（可观察浏览器窗口）
   - 自动识别页面元素，无需提供选择器
   - 结构化数据输出，可直接用于分析

### 备选（如京东反爬）
```
帮我打开百度搜索"AI Agent 框架"，采集前 10 条搜索结果的标题和链接
```

---

## 场景二：自我进化演示

**展示能力**：记忆系统 + 自我学习 + 知识图谱

### 演示步骤

1. 第一轮对话 — 建立记忆：

```
帮我记住：我们公司的服务器 IP 是 192.168.1.100，端口 22，用户名 admin
```

2. 第二轮对话 — 验证记忆：

```
我们的服务器地址是什么？
```
灵雀应从长期记忆中自动召回信息。

3. 第三轮 — 展示学习：

```
请写一个 Python 函数，计算斐波那契数列的第 n 项
```

如果第一次写的代码有性能问题，指出后灵雀会记录纠错配对，下次遇到类似问题会自动使用优化方案。

4. 展示知识图谱：

```
查询知识图谱中关于"服务器"的信息
```

### 演示要点
- 跨会话记忆持久化（即使重启也不丢失）
- 自动提取实体和关系（知识图谱）
- 错误-修复配对学习（自我进化）
- 经验自动召回（下次类似任务更快更好）

---

## 演示前准备

1. 确保服务已启动：`systemctl status lingque`
2. 确保浏览器可用：`google-chrome --version`
3. 飞书机器人已添加到演示群
4. `.env` 中 LLM API Key 已配置
5. 建议用非无头模式展示浏览器操作：`.env` 中设置 `BROWSER_HEADLESS=false`
