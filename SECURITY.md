# 安全设计文档

灵雀（LingQue）从架构设计层面实现了 **9 层纵深安全防护**，确保 AI Agent 在拥有强大能力的同时不会越权操作，所有数据完全存储在用户自己的服务器上。

---

## 安全架构概览

```
┌──────────────────────────────────────────────────────────┐
│  第 1 层  通道鉴权 — 用户白名单 + 消息签名 + 并发限流       │
├──────────────────────────────────────────────────────────┤
│  第 2 层  操作分级 — low / medium / high 三级风险管控       │
├──────────────────────────────────────────────────────────┤
│  第 3 层  文件隔离 — 路径白名单 + 敏感路径黑名单 + 防穿越    │
├──────────────────────────────────────────────────────────┤
│  第 4 层  命令安全 — 只读白名单 + 危险命令黑名单 + AST 审查  │
├──────────────────────────────────────────────────────────┤
│  第 5 层  密钥保护 — 全链路脱敏 + 凭证保险箱 + 日志脱敏      │
├──────────────────────────────────────────────────────────┤
│  第 6 层  沙箱隔离 — Docker 容器 + 资源限制 + 网络隔离       │
├──────────────────────────────────────────────────────────┤
│  第 7 层  会话安全 — contextvars 隔离 + 会话超时 + 磁盘持久化│
├──────────────────────────────────────────────────────────┤
│  第 8 层  网络安全 — WebSocket 长连接 + 无需公网端口暴露     │
├──────────────────────────────────────────────────────────┤
│  第 9 层  审计追溯 — 工具调用日志 + 安全审计日志 + 每日摘要   │
└──────────────────────────────────────────────────────────┘
```

## 第 1 层：通道鉴权

| 机制 | 说明 |
|------|------|
| 用户白名单 | `FEISHU_ALLOWED_USERS` 配置允许访问的飞书用户 open_id，未授权用户消息直接丢弃 |
| Webhook 签名 | 飞书事件回调通过 `verification_token` 验证来源真实性 |
| 并发限流 | `asyncio.Semaphore(3)` 限制同时处理的消息数，防止资源耗尽 |
| 消息去重 | 300 秒 TTL 的消息 ID 缓存，防止重复处理 |
| 启动检查 | 启动时检测空白名单并输出安全警告 |

## 第 2 层：操作分级

所有技能（Skill）注册时必须声明风险等级：

| 等级 | 行为 | 示例 |
|------|------|------|
| `low` | 静默执行 | 读文件、浏览网页、查询命令 |
| `medium` | 执行并记录审计日志 | 写文件、发送邮件 |
| `high` | 弹出飞书确认卡片，用户明确同意后才执行 | 删除文件、执行系统命令、清空记忆 |

高风险操作会在飞书中显示交互式确认卡片，包含操作详情和 Confirm/Cancel 按钮，60 秒超时自动取消。

## 第 3 层：文件隔离

```
ALLOWED_PATHS=./workspaces,/tmp
```

- **路径白名单**：所有文件操作必须在 `ALLOWED_PATHS` 范围内
- **路径解析**：使用 `Path.resolve()` 消除符号链接和 `..` 路径穿越
- **敏感路径黑名单**：即使在白名单目录内，以下文件也禁止直接读写：
  - `credentials.json`（凭证保险箱数据）
  - `cookies/`（浏览器登录状态）
  - `.chrome_profile/`（浏览器配置文件）
  - `.env`（环境变量配置）
- **大小限制**：读取 1MB、写入 50KB、目录列表 200 项

## 第 4 层：命令执行安全

### 只读命令（run_query）
使用白名单机制，仅允许安全的查看类命令：
- `ls`, `cat`, `grep`, `find`, `wc`, `head`, `tail`, `ps`, `df`, `free`, `uptime` 等

### 执行命令（run_command）
使用黑名单机制，拦截危险操作：
- **危险命令**：`rm -rf /`, `mkfs`, `curl | sh`, `passwd`, `iptables -F` 等
- **敏感路径**：包含 `.env`, `.ssh/`, `id_rsa`, `authorized_keys`, `api_key` 的命令被拦截
- **注入防护**：拦截 `$()`, 反引号, `<(` 等命令注入模式

### Python 代码执行（run_python）
使用 AST 语法树分析，拦截：
- 危险模块导入：`subprocess`, `os`, `sys`, `shutil` 等
- 危险内置函数：`exec`, `eval`, `compile`, `__import__`
- Dunder 属性访问：`__class__`, `__subclasses__` 等

## 第 5 层：密钥保护

### 全链路脱敏
20+ 条正则匹配模式，覆盖：
- OpenAI / Anthropic / DeepSeek / AWS 等 API Key
- Bearer Token / JWT / Private Key
- 飞书 App ID / Secret
- 通用 `key=value` 模式

脱敏点：最终用户回复、长期记忆写入、日志记录、飞书卡片发送、上下文压缩。

### 凭证保险箱
- 通过 `save_credential` / `get_credential` 安全存取第三方密钥
- 密钥以 Base64 编码存储在 `credentials.json`
- 启动时自动加载到 `os.environ`，运行时直接读取环境变量
- 工具参数日志中密钥值显示为 `***`

### .env 文件保护
- `.env` 文件在敏感路径黑名单中，Agent 无法通过文件操作读写
- 系统提示词明确禁止 Agent 将密钥写入 `.env` 或长期记忆

## 第 6 层：沙箱隔离

Docker 沙箱提供不可信代码的隔离执行环境：

| 限制 | 配置 |
|------|------|
| 内存 | 256MB |
| CPU | 50%（cpu_quota=50000） |
| 网络 | 默认禁用（network_disabled=True） |
| 生命周期 | 临时容器，执行后自动销毁 |
| 包安装 | 包名正则校验，防注入 |

## 第 7 层：会话安全

- **请求级隔离**：基于 `contextvars.ContextVar` 实现，不同群聊/用户的对话完全隔离
- **会话超时**：空闲 8 小时后自动压缩上下文（保留记忆，释放资源）
- **每日重置**：每日凌晨 4 点自动提取重要信息到长期记忆，清理会话
- **磁盘持久化**：会话状态持久化到磁盘，重启后自动恢复
- **Session ID 净化**：正则过滤特殊字符，防止路径注入

## 第 8 层：网络安全

- **WebSocket 长连接**：飞书通道使用 WebSocket 模式，服务器无需暴露公网端口
- **零公网暴露**：无需 Nginx、无需域名、无需 SSL 证书即可运行
- **CDP 本地连接**：浏览器自动化通过本地 CDP 端口连接，不暴露远程调试接口

## 第 9 层：审计追溯

- **工具调用日志**：每次技能执行记录名称、参数（敏感参数脱敏）、耗时、结果
- **安全审计日志**：高危操作写入独立 `logs/security_audit.log`
- **消息日志**：每日 JSONL 格式日志，记录对话摘要（内容截断 + 脱敏）
- **每日摘要**：自动生成当日工作摘要，可通过飞书推送

## 数据隐私与合规

### 数据本地化
- **所有数据存储在用户自己的服务器上**，不上传任何第三方平台
- 对话记录、长期记忆、知识图谱、浏览器 Cookie、凭证 —— 全部本地存储
- 唯一的外部通信是与 LLM API 的交互（必需）和飞书/钉钉消息通道

### 个人信息保护
- 符合《中华人民共和国个人信息保护法》的"最小必要"原则
- 仅收集完成任务所需的最少信息
- 用户可随时通过 `/clear` 命令清空会话数据
- 长期记忆可通过 `memory_clear` 技能完全删除

### 数据生命周期
| 数据类型 | 存储位置 | 保留策略 |
|---------|---------|---------|
| 会话消息 | 内存 + 磁盘 | 空闲 8h 压缩，每日重置 |
| 长期记忆 | `memory/MEMORY.md` | 持久保存，用户可删除 |
| 知识图谱 | `memory/knowledge.db` | 持久保存，用户可删除 |
| 学习数据 | `memory/learning.db` | 持久保存，用户可删除 |
| 浏览器 Cookie | `workspaces/cookies/` | 按域名存储，用户可删除 |
| 凭证 | `workspaces/credentials.json` | 加密存储，用户可删除 |
| 操作日志 | `memory/daily/` | 按日存储 |

## 安全配置建议

### 生产环境必须配置
```ini
# 限制飞书用户访问
FEISHU_ALLOWED_USERS=ou_xxx,ou_yyy

# 限制文件操作范围
ALLOWED_PATHS=/www/wwwroot/lingque/workspaces,/tmp

# 开启高危操作确认
REQUIRE_CONFIRMATION=true
```

### 可选加固
```ini
# Docker 沙箱（需安装 Docker）
DOCKER_SANDBOX_ENABLED=true

# 关闭向量记忆（减少攻击面）
VECTOR_MEMORY_ENABLED=false
```
